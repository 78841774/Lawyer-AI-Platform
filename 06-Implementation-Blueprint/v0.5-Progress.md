# v0.5 MVP Coding Progress

## 已完成内容

* 创建 Lawyer-AI-Platform-App 项目代码骨架。
* 创建 frontend 目录，采用 Next.js、TypeScript、Tailwind 技术栈。
* 创建 Workspace 页面骨架，包括 Dashboard、Case List、Case Detail、Material Upload、Fact View、Legal Analysis View、Report View。
* 创建 backend 目录，采用 FastAPI 和 Python 3.11。
* 实现 GET /health，返回 {"status":"ok"}。
* 创建 infra/database.sql，定义 PostgreSQL MVP 表：cases、materials、facts、legal_issues、reports。
* 创建 storage 目录，包含 original-files、ocr-files、reports 三类存储路径。
* 创建 ai-runtime 目录，包含 fact-runtime、legal-runtime、report-runtime 和 runtime interface。
* 创建 docker-compose.yml，包含 frontend、backend、postgres 服务。
* v0.6 已集成 Case Service 到主后端，支持案件创建与查询。
* v0.7 已集成 Material Service 到主后端，支持案件材料上传与查询。
* v0.8 已将 Case Service 和 Material Service 从内存 dict 迁移到 SQLite + SQLAlchemy。
* v0.9 已实现 Fact Runtime，支持从已上传材料中按规则提取案件事实，并将 facts 持久化到 SQLite。
* v1.0 已实现 Legal Analysis Runtime，支持基于已抽取 facts 生成规则法律分析，并将 legal analyses 持久化到 SQLite。
* v1.1 已实现 Report Runtime，支持基于 facts 和最新 legal analysis 生成初步报告，并保存到 reports 表和 storage/reports。
* v1.2 已实现 Workspace API，支持前端查询案件、报告详情和 dashboard 统计数据。
* v1.3 已实现 Frontend Workspace Dashboard，支持首页统计、案件列表、报告列表和报告详情页面。
* v1.4 已实现 Demo Flow Polish，支持在浏览器中创建案件、上传材料、抽取事实、运行法律分析并生成报告。
* v1.5 已完成 Demo Stabilization，新增本地启动脚本、端口清理脚本和 Demo Runbook，并删除不再使用的旧 07-Case-Service 目录。
* v2.0 已完成 Skill Training Specification，定义 Skill Factory、Skill Schema、训练流水线、评估框架、打包系统、版本策略和生命周期。
* v2.1 已实现 Skill Training Runtime 初版，支持从已有 case、facts、legal analysis 和 reports 生成规则式 Skill Candidate 草稿，保存到 skills 表并导出本地 skill package。
* v2.2 已实现 Skill Evaluation Runtime 初版，支持对 Skill Candidate 进行六维规则评估，写入 evaluation_details、validation_status 和本地 skill.json。
* v2.3 已实现 Experience Package Builder 初版，支持将 validated skill 构建为本地 Experience Package，写入 experience_packages 表并生成 package.json manifest。
* v2.4 已实现 Skill Registry 初版，支持聚合 skills 与 experience_packages，发布 validated/built 能力并执行 deprecated 生命周期状态流转。
* v2.5 已实现 Workspace Skill Loader 初版，支持 Workspace 查询 published Skills、案件应用 Skill，并在前端展示 Available Skills 与 Applied Skills。
* v2.6 已完成 Skill-Aware Runtime 闭环，支持已应用 Skill 的 Fact、Legal Analysis、Report 后端处理及前端展示。
* v2.7-A 已新增统一 LLM Adapter 抽象层，提供 mock/openai client、PromptContextBuilder 与 `/llm/status`、`/llm/test` 接口。
* v2.8 已完成 DeepSeek Live Mode，支持通过 `LLM_PROVIDER=deepseek` 调用 DeepSeek OpenAI-compatible API，并保持 mock 默认模式。
* v2.9 已完成 Production Readiness Foundation，新增 `APP_ENV`、PostgreSQL `DATABASE_URL` 支持、Alembic 基础结构和 Docker Compose PostgreSQL 配置。
* v3.0 已进入 Internal Alpha Foundation，新增 Local Demo User、Local Demo Workspace、workspace member、案件归属字段和 workspace 案件 API。
* v3.1 已完成 Auth Foundation，新增 Dev Token/API Key 请求身份识别、`auth_tokens` hash 存储、`/auth/status` 与核心 users/workspaces/cases current-user 依赖。
