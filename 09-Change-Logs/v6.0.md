# v6.0 Personal Alpha Case OS Foundation

## Summary

Added a local-only Personal Alpha Case OS foundation for aggregating controlled v5.0-v5.9 metadata into a single case-level view.

## Backend

- Added `personal_alpha_case_os` backend module.
- Added Case OS status, case list, detail, audit timeline, next action, and safety checklist APIs.
- Added metadata-only state machine and next-action derivation.
- Added audit timeline aggregation from Personal Alpha controlled workflow metadata.
- Registered `/case-os` routes in `app/main.py`.

## Frontend

- Added `/case-os` case list page.
- Added `/case-os/[caseId]` case detail page.
- Added Personal Alpha Case OS navigation entry in AppShell.
- Added Case OS API client functions and TypeScript types.

## APIs

- `GET /case-os/status`
- `GET /case-os`
- `GET /case-os/{case_id}`
- `GET /case-os/{case_id}/audit-timeline`
- `GET /case-os/{case_id}/next-action`
- `GET /case-os/{case_id}/safety-checklist`

## Safety Boundary

- local-only
- mock-first
- controlled-first
- metadata-only
- redacted-only
- preview-only
- advisory-only
- manual-review-required
- lawyer-review-required

No real LLM, DeepSeek live, real OCR, legal database, raw material text, raw OCR text, raw legal-search result text, raw quote, final legal opinion, final report body, Skill publishing, or Workspace Runtime activation is enabled.

## Notes

Case OS aggregates ignored runtime metadata and does not store raw content in Git-tracked files.
