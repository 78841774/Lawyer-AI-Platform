# v5.1 Personal Alpha Case Dashboard

## Summary

Added an independent Personal Alpha Case Dashboard for the v5.0 local Personal Alpha workspace. The dashboard aggregates mock and redacted workspace metadata into a local-only control surface.

Baseline:

- Previous version: `v5.0-personal-alpha-workspace`
- v5.1 commit: `7ed843e`
- Commit message: `feat: add personal alpha case dashboard v5.1`

## Added

Backend:

- Added `personal_alpha_dashboard` module.
- Added dashboard schema models for status, workspace summary, stage health, audit timeline, and source trace summary.
- Added dashboard engine that aggregates v5.0 workspace runtime metadata.
- Registered `/personal-alpha-dashboard` routes in `app/main.py`.

Frontend:

- Added `/personal-alpha-dashboard` page.
- Added Safety Boundary card.
- Added Workspace Summary cards.
- Added seven Stage Health cards.
- Added Audit Timeline JSON panel.
- Added Source Trace Summary JSON panel.
- Added AppShell navigation entry for `Personal Alpha Dashboard`.
- Added dashboard API helpers in `frontend/services/api.ts`.
- Added dashboard TypeScript types in `frontend/types/index.ts`.

## APIs

Added:

- `GET /personal-alpha-dashboard/status`
- `GET /personal-alpha-dashboard/summary`
- `GET /personal-alpha-dashboard/stage-health`
- `GET /personal-alpha-dashboard/audit-timeline`
- `GET /personal-alpha-dashboard/source-trace-summary`

## Changed

- Extended the Personal Alpha frontend navigation to include the dashboard page.
- Extended the frontend API layer with dashboard request helpers.
- Extended frontend types with dashboard response shapes.
- Extended backend routing to include the new dashboard router.

## Fixed

- No behavioral fix was required in v5.1. The release adds a dashboard layer without changing the v5.0 workflow execution model.

## Security

v5.1 preserves the Personal Alpha safety boundary:

- local-only
- mock-first
- controlled-first
- metadata-only
- redacted-only
- advisory-only
- manual-review-required

Provider and production behavior:

- `production_enabled=false`
- `llm_live_enabled=false`
- `deepseek_live_enabled=false`
- `ocr_live_enabled=false`
- `legal_search_live_enabled=false`
- `final_legal_opinion_enabled=false`
- `auto_skill_publish_enabled=false`
- `auto_workspace_runtime_enabled=false`

Content boundary:

- no raw material text returned
- no raw OCR text returned
- no raw legal search result text returned
- no raw source quote text returned
- no final legal opinion generated
- no final report body generated
- no real provider calls
- no raw content written into Git-tracked paths
