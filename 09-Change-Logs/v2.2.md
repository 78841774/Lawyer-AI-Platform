# v2.2 Skill Evaluation Runtime

## Scope

v2.2 adds Skill Evaluation Runtime to the main backend.

The platform can now evaluate a Skill Candidate and mark it as `validated` or `needs_improvement`.

## Added

* Added `app/skill_training/evaluation_metrics.py`.
* Added `app/skill_training/evaluation_engine.py`.
* Added `app/skill_training/evaluation_report_builder.py`.
* Added `POST /skills/{skill_id}/evaluate`.
* Added `GET /skills/{skill_id}/evaluation`.
* Added `evaluation_details`, `validation_status`, and `validated_at` to the Skill model.
* Added local Skill Package `skill.json` update after evaluation.

## Metrics

The runtime evaluates six dimensions:

* fact_pattern_quality
* reasoning_quality
* prompt_quality
* template_quality
* legal_relevance
* report_reusability

The final score is the average of all six metrics.

## Validation Rule

```text
evaluation_score >= 0.75 -> validated
evaluation_score < 0.75  -> needs_improvement
```

## Package Update

After evaluation, the runtime updates:

```text
Lawyer-AI-Platform-App/skills/{skill_id}/skill.json
```

with:

* evaluation_score
* validation_status
* evaluation_details

## Database Note

Local SQLite uses a startup compatibility check to add missing v2.2 columns.

Production should use Alembic migration for the same schema change.
