# v1.3 Frontend Workspace Dashboard

## Scope

* Connected the existing Next.js frontend to the v1.2 Workspace API.
* Added a real dashboard at `/` with workspace stats.
* Updated `/cases` to render persisted cases from `GET /cases`.
* Added `/reports` to render persisted reports from `GET /reports`.
* Added `/reports/[report_id]` to render report detail from `GET /reports/{report_id}`.
* Updated `frontend/services/api.ts` with Workspace API clients and response types.

## Behavior

* The frontend API base URL defaults to `http://127.0.0.1:8001`.
* Pages use dynamic server rendering so local builds do not require a live backend.
* Pages show a compact unavailable state when the backend is not running.

## Verification

```bash
cd Lawyer-AI-Platform-App/frontend
npm install
npm run build
```

With the backend running on port 8001, open:

```text
http://localhost:3000
```
