# v2.9 Production Readiness Foundation

## Added

* Added `APP_ENV` configuration with `local` as the default.
* Kept `DATABASE_URL=sqlite:///./local.db` as the default local database.
* Added PostgreSQL support through `psycopg2-binary`.
* Added Alembic base structure under `backend/alembic`.
* Added `backend/alembic.ini`.
* Added Docker Compose PostgreSQL configuration for production-style local testing.
* Added `Production-Readiness.md`.

## Changed

* Backend startup now runs SQLAlchemy `create_all` only when `APP_ENV=local`.
* `APP_ENV=production` expects schema to be managed by Alembic migrations.
* `.env.example` now documents local, LLM, and PostgreSQL environment variables without real secrets.
* Local development and demo docs now document local SQLite mode, PostgreSQL mode, and Alembic checks.

## Notes

* v2.9 does not add user login.
* v2.9 does not perform cloud deployment.
* v2.9 does not generate a full initial Alembic migration yet.
* Before v3.0, generate and review the initial migration from the current SQLAlchemy models.
