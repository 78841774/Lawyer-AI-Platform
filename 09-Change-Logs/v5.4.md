# v5.4 Personal Alpha Source Review Decision

## Added

- Added source-review decision engine for Personal Alpha metadata review.
- Added ignored runtime decision storage under `storage/runtime/personal_alpha_source_review/decisions`.
- Added decision APIs:
  - `GET /personal-alpha-source-review/run/{workspace_run_id}/decisions`
  - `POST /personal-alpha-source-review/run/{workspace_run_id}/decisions`
  - `GET /personal-alpha-source-review/run/{workspace_run_id}/decision-summary`
- Added frontend decision form, decision summary, and decision history on `/personal-alpha-source-review`.
- Added TypeScript API helpers and types for decision list, decision submit, and decision summary.

## Safety

- Decisions are advisory metadata only.
- Allowed decision values are `approve`, `reject`, `request_revision`, and `mark_unclear`.
- Invalid decisions return `invalid_decision` safely.
- Manual review and metadata-only gates are required before storing decisions.
- Unsafe raw-content-like payloads are blocked and not written to runtime storage.
- No raw material text, raw OCR text, raw legal search results, local paths, API keys, or formal legal opinion content are returned or stored.

## Validation Scope

- Backend compile validation.
- Frontend production build.
- Status, run detail, source traces, evidence summary, decisions, decision submit, and decision summary API checks.
- Invalid decision, unsafe reason, and not-found checks.
- Existing v5.0, v5.1, v5.2, and v5.3 API regression checks.
