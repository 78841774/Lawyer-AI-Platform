# v6.1 Personal Alpha Case OS Stage Orchestrator

## Objective

v6.1 enhances the v6.0 Personal Alpha Case OS foundation with a stable stage orchestration layer for current stage, next action, blockers, route mapping, transition preview, and action eligibility.

## Relationship to v6.0

v6.0 created the Case OS foundation and metadata aggregation endpoints. v6.1 keeps that foundation intact and adds orchestration endpoints and frontend panels that consume the same metadata-only workflow state.

## Stage Orchestration API

Added:

- `GET /case-os/{case_id}/stage-orchestration`
- `GET /case-os/{case_id}/stage-transitions`
- `GET /case-os/{case_id}/action-eligibility`
- `GET /case-os/{case_id}/blockers`

## Stage Transition Preview

The transition preview reports completed, available, pending, or blocked transitions between ordered Case OS stages. It is advisory only and does not execute workflow actions.

## Action Eligibility Logic

Action eligibility now evaluates:

- current Case OS next action
- existing metadata prerequisites
- lawyer final review state
- manual review confirmations
- metadata-only confirmations
- no final legal opinion and no final report generation confirmations

## Blocker Summary Logic

The blocker summary aggregates fixed, metadata-only blocked reasons at top level and per stage. It does not include raw material content, raw OCR text, raw legal search text, local paths, or filenames.

## Route Mapping

Stage and action routes are centralized in `personal_alpha_case_os/stage_routes.py`. The mapping returns frontend paths only and uses metadata identifiers where needed.

## Frontend Updates

The Case OS list page now shows current stage, next action, blocked state, blocked reasons, and explicit navigation actions.

The Case OS detail page now shows Stage Orchestration, Stage Progress, Action Eligibility, Blockers, Stage Transitions, and JSON panels for the new v6.1 responses while preserving Case Profile, Next Action, Stage Summary, Audit Timeline, and Safety Checklist.

## Metadata-Only Rule

All v6.1 responses are metadata-only and include `mock_or_redacted_only=true` and `raw_content_included=false`.

## No Raw Content Rule

Unsafe or path-like case ids are blocked or redacted without echoing local paths or filenames.

## No Final Legal Opinion

v6.1 does not generate formal legal opinions and reports `final_legal_opinion_generated=false`.

## No Final Report Generated

v6.1 does not generate final report bodies and reports `final_report_generated=false`.

## v6.2 Readiness

v6.1 prepares for a unified Case OS audit timeline by stabilizing stage ids, action ids, route mapping, blocker summaries, and eligibility metadata.
