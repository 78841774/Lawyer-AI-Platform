# v2.4 Skill Registry

## Scope

v2.4 adds Skill Registry to the main backend.

The registry provides a unified view of Skills and Experience Packages for future Workspace capability loading.

## Added

* Added `app/api/skill_registry.py`.
* Added `app/services/skill_registry_service.py`.
* Added `app/repositories/skill_registry_repository.py`.
* Added `app/models/skill_registry.py`.
* Added `GET /skill-registry`.
* Added `GET /skill-registry/{skill_id}`.
* Added `GET /skill-registry/domains`.
* Added `POST /skill-registry/{skill_id}/publish`.
* Added `POST /skill-registry/{skill_id}/deprecate`.

## Registry Storage

v2.4 does not create a new registry table.

The registry is an aggregation service:

* `skills` remains the source of truth for skill metadata, validation status, evaluation score, and lifecycle status.
* `experience_packages` remains the source of truth for package metadata and package status.

## Publish Rule

Publishing requires:

```text
skill.validation_status = validated
package.status = built
```

When published:

```text
skill.status = published
package.status = published
```

## Deprecation Rule

When deprecated:

```text
skill.status = deprecated
package.status = deprecated
```

Deprecated skills cannot be published again.
