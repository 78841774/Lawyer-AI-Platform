# v2.8 DeepSeek Live Mode

## Added

* Added `DeepSeekLLMClient` for real DeepSeek API calls through the OpenAI-compatible SDK.
* Added `LLM_PROVIDER=deepseek` runtime selection while keeping `mock` as the default provider.
* Added `DEEPSEEK_API_KEY`, `DEEPSEEK_MODEL`, `DEEPSEEK_BASE_URL`, and `DEEPSEEK_TIMEOUT_SECONDS` configuration.
* Enhanced `GET /llm/status` to return `provider`, `model`, `configured`, and `base_url_configured`.
* `POST /llm/test` continues to accept `prompt` and optional `context`, and returns the unified LLM response shape.

## Defaults

* `DEEPSEEK_MODEL=deepseek-chat`
* `DEEPSEEK_BASE_URL=https://api.deepseek.com`
* `DEEPSEEK_TIMEOUT_SECONDS=30`
* `LLM_PROVIDER=mock`

## Error Handling

* Mock mode never checks `DEEPSEEK_API_KEY`.
* DeepSeek mode starts safely without `DEEPSEEK_API_KEY`.
* Missing DeepSeek API keys return `status=error` with `DEEPSEEK_API_KEY not configured`.
* DeepSeek request failures return the unified error response with zero token usage and sanitized error text.

## Verification

* `python -m compileall app`
* Mock mode `GET /llm/status`
* Mock mode `POST /llm/test`
* DeepSeek mode without API key `GET /llm/status`
* DeepSeek mode without API key `POST /llm/test`
