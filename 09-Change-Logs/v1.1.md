# v1.1 Report Runtime

## Scope

* Added Report Runtime to the main backend at `Lawyer-AI-Platform-App/backend`.
* Added the `reports` SQLite table through SQLAlchemy startup `create_all`.
* Added preliminary Markdown report generation from persisted facts and the latest legal analysis.
* Added API endpoint:
  * `POST /cases/{case_id}/reports/generate`

## Report Sections

* Executive Summary
* Facts Summary
* Legal Issues
* Legal Analysis
* Preliminary Conclusion

## Storage

* Report metadata and content are saved to the `reports` table.
* Markdown files are saved under `Lawyer-AI-Platform-App/storage/reports/{case_id}/`.

## Behavior

* Missing cases return `404`.
* Cases without facts return `400` and instruct the caller to run Fact Runtime first.
* Cases without legal analysis return `400` and instruct the caller to run Legal Analysis Runtime first.
