# v2.3 Experience Package Builder

## Scope

v2.3 adds Experience Package Builder to the main backend.

The platform can now turn a validated Skill Candidate into a standard local Experience Package.

## Added

* Added `ExperiencePackage` model and `experience_packages` table.
* Added `ExperiencePackageRepository`.
* Added `ExperiencePackageService`.
* Added `app/api/experience_packages.py`.
* Added `app/skill_training/package_builder.py`.
* Added `app/skill_training/package_validator.py`.
* Added `app/skill_training/package_manifest.py`.
* Added `POST /skills/{skill_id}/packages/build`.
* Added `GET /experience-packages`.
* Added `GET /experience-packages/{package_id}`.
* Added `GET /experience-packages/{package_id}/manifest`.

## Build Rule

The builder requires:

* skill exists
* skill has evaluation details
* validation_status is `validated`

## Package Output

```text
Lawyer-AI-Platform-App/experience-packages/{package_id}/
├── package.json
├── skill.json
├── prompts/
│   ├── fact_prompt.txt
│   ├── analysis_prompt.txt
│   └── report_prompt.txt
├── templates/
│   └── report_template.md
└── tests/
    └── test_case.json
```

## Notes

v2.3 package status is `built`.

`built` does not mean `published`.

Package Registry, release approval, and Workspace loading remain future work.
