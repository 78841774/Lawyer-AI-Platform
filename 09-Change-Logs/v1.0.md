# v1.0 Legal Analysis Runtime

## Scope

* Added Legal Analysis Runtime to the main backend at `Lawyer-AI-Platform-App/backend`.
* Added the `legal_analyses` SQLite table through SQLAlchemy startup `create_all`.
* Added rule-based legal analysis generation from persisted facts.
* Added API endpoints:
  * `POST /cases/{case_id}/analysis/run`
  * `GET /cases/{case_id}/analysis`

## Behavior

* Missing cases return `404`.
* Cases without facts return `400` and instruct the caller to run Fact Runtime first.
* Analysis output is saved to the database with JSON-string fields for `issues`, `rules`, and `reasoning`.
* The list endpoint returns all analysis records for the case in creation order.

## Verification

Run the backend from `Lawyer-AI-Platform-App/backend`, then use:

```bash
curl http://127.0.0.1:8001/health
curl http://127.0.0.1:8001/cases/case_001/facts
curl -X POST http://127.0.0.1:8001/cases/case_001/analysis/run
curl http://127.0.0.1:8001/cases/case_001/analysis
```

Restart the backend and run the final analysis query again to confirm persistence.
