# v0.9 Fact Runtime

## Scope

* Added Fact Runtime to the main backend at `Lawyer-AI-Platform-App/backend`.
* Added the `facts` SQLite table through SQLAlchemy startup `create_all`.
* Added rule-based fact extraction from uploaded material files.
* Added API endpoints:
  * `POST /cases/{case_id}/facts/extract`
  * `GET /cases/{case_id}/facts`

## Behavior

* Missing cases return `404`.
* Cases without materials return an empty `facts` list.
* Missing material files are recorded as skipped facts with `status="skipped"`.
* Text material content is split by non-empty lines when possible; otherwise the whole file is saved as one fact.

## Verification

Run the backend from `Lawyer-AI-Platform-App/backend`, then use:

```bash
curl http://127.0.0.1:8001/health
curl http://127.0.0.1:8001/cases/case_001
curl http://127.0.0.1:8001/cases/case_001/materials
curl -X POST http://127.0.0.1:8001/cases/case_001/facts/extract
curl http://127.0.0.1:8001/cases/case_001/facts
```

Restart the backend and run the final facts query again to confirm persistence.
