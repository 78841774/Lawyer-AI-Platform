# v2.1 Skill Training Runtime

## Scope

v2.1 adds the first Skill Training Runtime to the main backend.

The platform can now move from processing a case to mining a case for a Skill Candidate draft.

## Added

* Added `app/skill_training/` runtime components.
* Added `Skill` model and `skills` table.
* Added `SkillRepository` and `SkillService`.
* Added `app/api/skills.py`.
* Added `POST /cases/{case_id}/skills/build`.
* Added `GET /skills`.
* Added `GET /skills/{skill_id}`.
* Added local package export under `Lawyer-AI-Platform-App/skills/{skill_id}/`.

## Runtime Flow

```text
Case
  |
  v
Facts
  |
  v
Legal Analysis
  |
  v
Reports
  |
  v
Skill Candidate
  |
  v
Local Skill Package
```

## Package Output

```text
Lawyer-AI-Platform-App/skills/{skill_id}/
├── skill.json
├── fact_prompt.txt
├── analysis_prompt.txt
├── report_prompt.txt
└── templates/
    └── report_template.md
```

## Error Handling

* Missing case returns `404`.
* Missing facts returns `400`.
* Missing legal analysis returns `400`.
* Missing reports returns `400`.

## Notes

v2.1 uses rule-based generation only.

No real AI runtime is called in this version.
