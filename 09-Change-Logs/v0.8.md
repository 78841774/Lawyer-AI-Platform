# Change Log v0.8

## Database Integration

将 v0.6 Case Service 和 v0.7 Material Service 从进程内 dict 存储迁移到 SQLite + SQLAlchemy。

新增和更新内容：

* 新增 backend/app/core/database.py，提供 SQLite engine、SessionLocal、get_db 和启动建表逻辑。
* 更新 backend/app/models/case.py，定义 SQLAlchemy cases 表模型。
* 更新 backend/app/models/material.py，定义 SQLAlchemy materials 表模型。
* 新增 backend/app/repositories/case_repository.py。
* 新增 backend/app/repositories/material_repository.py。
* 更新 backend/app/services/case_service.py，改为通过 repository 访问数据库。
* 更新 backend/app/services/material_service.py，改为保存文件后写入数据库。
* 更新 backend/app/api/cases.py，保持 POST /cases 和 GET /cases/{case_id} 接口兼容。
* 更新 backend/app/api/materials.py，保持 POST /cases/{case_id}/materials 和 GET /cases/{case_id}/materials 接口兼容。
* 更新 backend/app/main.py，启动时自动 create_all。
* 更新 backend/requirements.txt，增加 SQLAlchemy。
* 更新 Local Development 文档，增加 SQLite 本地开发与重启持久化验证说明。

本地数据库文件：

```text
Lawyer-AI-Platform-App/backend/local.db
```

验证命令：

* curl http://127.0.0.1:8001/health
* curl -X POST http://127.0.0.1:8001/cases -H "Content-Type: application/json" -d '{"title":"数据库测试案件"}'
* echo "db test material" > /tmp/db-test-material.txt
* curl -X POST "http://127.0.0.1:8001/cases/case_001/materials" -F "file=@/tmp/db-test-material.txt" -F "material_type=document"
* curl http://127.0.0.1:8001/cases/case_001
* curl http://127.0.0.1:8001/cases/case_001/materials
* 重启后再次查询 case_001 和 materials，确认 SQLite 数据不会丢失。
