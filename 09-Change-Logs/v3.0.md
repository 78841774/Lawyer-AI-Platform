# v3.0 Internal Alpha Foundation

## Added

* Added `users`, `workspaces`, and `workspace_members` SQLAlchemy models.
* Added Local Demo Identity seeding for `APP_ENV=local`.
* Added `GET /users/me`.
* Added workspace APIs:
  * `GET /workspaces`
  * `GET /workspaces/{workspace_id}`
  * `GET /workspaces/{workspace_id}/cases`
  * `POST /workspaces/{workspace_id}/cases`
* Added `Internal-Alpha.md`.

## Changed

* Added `workspace_id` and `owner_user_id` to `cases`.
* `POST /cases` now assigns the default local workspace and local user.
* `GET /cases` and `GET /cases/{case_id}` now return case ownership fields.
* Dashboard now displays Current User and Current Workspace.
* Case list and case detail views now display workspace and owner identifiers.
* Backend app version is now `3.0.0`.

## Notes

* v3.0 does not implement real login.
* v3.0 does not implement JWT authentication.
* v3.0 does not implement payment or complex RBAC.
* SQLite local startup can add missing case ownership columns for MVP compatibility.
* Production schema changes should use Alembic migrations.
* v3.1 should introduce the real authentication system.
