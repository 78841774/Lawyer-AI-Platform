# v7.31g Practice Runtime Controlled Loading & Monitoring

## Added

- Added Practice Runtime Loading Registry metadata for lawyer-approved v7.31f experience packages.
- Added controlled runtime load API with disabled, gray, active, blocked, disabled, and rolled-back states.
- Added scope controls for case cause, workspace, runtime mode, task type, rollout percentage, usage limit, and emergency disable.
- Added policy evaluation API that records allowed or blocked usage metadata.
- Added usage monitoring, risk event metadata, runtime audit, and runtime source trace endpoints.
- Added disable, gray enable, active enable, and rollback controls.
- Added v7.31g documentation and API regression script.

## Safety

- Only v7.31f `approved_for_practice_load` lawyer-approved packages can be loaded.
- Runtime loading reads and returns metadata only; source materials and source content are not returned.
- Blocked package load attempts are recorded as blocked metadata, not loaded runtime state.
- Disable and rollback preserve package, audit, and source trace records.
- No provider call, API key read, real training, formal training set write, Skill publish, final legal opinion, final report, public link, email, or external delivery is performed.
