# v1.2 Workspace API

## Scope

* Added Workspace API read endpoints for the frontend workspace.
* Added repository count/list helpers for cases, materials, facts, legal analyses, and reports.
* Added a Workspace service to aggregate dashboard data from SQLite.
* Registered Workspace API routes in the main backend.

## Endpoints

* `GET /cases`
* `GET /reports`
* `GET /reports/{report_id}`
* `GET /dashboard/stats`

## Dashboard Stats

The dashboard stats endpoint returns aggregate counts:

```json
{
  "cases": 1,
  "materials": 1,
  "facts": 2,
  "analyses": 1,
  "reports": 2
}
```

## Behavior

* `GET /reports/{report_id}` returns `404` when the report does not exist.
* All endpoints read from the existing SQLite database.
