# v3.1 Auth Foundation

## Added

* Added `auth_tokens` SQLAlchemy model.
* Added SHA-256 token hash storage.
* Added local dev token seeding for `APP_ENV=local`.
* Added `app/core/auth.py` for current-user resolution.
* Added `GET /auth/status`.
* Added `GET /auth/dev-token`.
* Added `Auth-Foundation.md`.

## Changed

* Backend app version is now `3.1.0`.
* `GET /users/me` now resolves the request user from auth context.
* Workspace APIs now use the authenticated current user.
* Case list, case detail, and case create APIs now use the authenticated current user.
* `POST /cases` now uses the current user's first active workspace.
* Dashboard now displays Auth Status and the current user auth source.

## Notes

* v3.1 is not a formal login system.
* v3.1 does not implement passwords, OAuth, or JWT.
* Plaintext tokens are not stored in the database.
* `APP_ENV=local` can fall back to `user_local_001` when no token is provided.
* Non-local environments require a token for protected endpoints.
* v3.2 can add JWT or OAuth on top of the current-user dependency.
