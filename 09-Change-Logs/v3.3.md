# v3.3 Frontend Auth Shell

## v3.3-A

feat: add frontend auth shell v3.3

## Added

* Dashboard auth card now resolves current user, current workspace, and current auth mode from authenticated API calls.
* Local Login continues to call `POST /auth/login` and stores the returned JWT in browser `localStorage`.
* Logout clears the stored JWT and refreshes auth state.
* Frontend API requests attach `Authorization: Bearer <JWT>` when a JWT is stored.
* Local API requests can fall back to `X-Dev-Token` when no JWT is stored.
* Cases list has a local mock fallback when the local API is unavailable.

## Notes

* Backend API, database, and authentication logic were not changed in this frontend shell phase.
* Dev token fallback is for local development only.
* v3.3 can continue toward a production auth provider, invite flow, and workspace member management.

## v3.3-B

feat: improve workspace case UI v3.3

## Added

* Dashboard shows a current-workspace cases area.
* Cases page displays current user, current workspace, `case_id`, `workspace_id`, and `owner_user_id`.
* New Case page displays current user and current workspace before creating a case.
* Case Detail page includes a Case ownership section with `workspace_id`, `owner_user_id`, and `created_at`.
* Workspaces page provides a view-cases entry for each workspace.

## Notes

* No backend authentication logic was changed.
* No Skill Training APIs or `/skill-candidates/*` APIs were added.
* Case creation continues to use the existing frontend API client and JWT authorization behavior.
