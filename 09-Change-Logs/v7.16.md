# v7.16 Controlled Case Analysis Runtime

## Added

- New backend module `personal_case_analysis`.
- New frontend page `/personal-case-analysis`.
- New controlled case analysis API prefix `/personal-case-analysis`.
- Skill baseline loader that references v7.15 Skill Training Runtime metadata.
- Fact Analysis Stage draft metadata.
- Legal Analysis Stage draft metadata.
- Review Queue, Source Trace, Audit, Safety, Evaluation, and Gate metadata.
- Personal Production runtime registry integration.
- Personal Production provider/capability summary integration.
- AppShell navigation entry.
- Lightweight regression script `check_personal_case_analysis_apis.sh`.
- Documentation `docs/Controlled-Case-Analysis-Runtime-v7.16.md`.

## Safety

- v7.16 is open-case execution, not closed-case training.
- Training data generation remains disabled.
- The runtime does not write to training sets.
- The runtime does not update or publish Skills.
- Evaluation and gate are reference-only.
- Gate does not block the next stage.
- Raw content and raw OCR text are not returned.
- API keys are not read or returned.
- No final legal opinion, final report, email, real PDF/DOCX, or external delivery is generated.

## Notes

- If v7.15 Skill baseline metadata is incomplete, v7.16 reports missing baseline metadata and uses placeholder lineage. It does not pretend a complete Skill exists.
- v7.16 remains part of the uncommitted v7.10-v7.17 Personal Live Intelligence & Controlled Case Analysis large stage.
