# v3.4 Change Log

## v3.4-A Real Case Intake Foundation

Implemented the first real-case intake foundation for AIHome.law.

### Added

* Case model now supports nullable intake fields:
  * `client_name`
  * `counterparty_name`
  * `contract_type`
  * `dispute_amount`
  * `jurisdiction`
  * `intake_notes`
* Existing `case_type` and `objective` fields are reused in the intake flow.
* Local SQLite startup compatibility adds missing case intake columns with `ALTER TABLE`.
* `POST /cases` accepts intake fields while remaining compatible with title-only creation.
* `GET /cases` and `GET /cases/{case_id}` return intake fields.
* Create Case page now provides a Chinese Real Case Intake form.
* Cases list displays client, counterparty, case type, contract type, and dispute amount.
* Case Detail displays an `Intake 信息` card.

### Not Included

* No complex OCR.
* No PDF parsing enhancement.
* No formal legal review workflow.
* No team approval workflow.
* No Skill Training main-chain changes.
* No `/skill-candidates/*` API.
* No production deployment changes.

## v3.4-B Folder-Aware Material Intake

Added folder-aware material intake for case detail workflows.

### Added

* Materials now support nullable path metadata:
  * `original_filename`
  * `relative_path`
  * `folder_path`
  * `file_ext`
  * `upload_batch_id`
  * `display_order`
* Local SQLite startup compatibility adds missing material path columns with `ALTER TABLE`.
* `POST /cases/{case_id}/materials` accepts optional `relative_path` and `upload_batch_id`.
* `POST /cases/{case_id}/materials/batch` accepts multiple files and per-file `relative_path` values.
* Material storage is constrained to `storage_root/original-files/{case_id}/{upload_batch_id}/`.
* `GET /cases/{case_id}/materials` returns folder-aware metadata with fallbacks for old rows.
* Case Detail material center supports single-file upload and browser folder upload through `webkitdirectory`.
* Materials are grouped by `folder_path` / `relative_path` in the UI.
* Fact extraction prompt and runtime context include material path metadata.
* Fact extraction responses include `source_refs.material_id`, `source_refs.filename`, and `source_refs.relative_path` when available.

### Not Included

* No complex OCR.
* No zip automatic extraction.
* No PDF content parsing enhancement.
* No Skill Training main-chain changes.
* No real case material committed to the repository.
