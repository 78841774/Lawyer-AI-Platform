# v3.2 JWT Auth Foundation

## Added

* Added JWT configuration: `JWT_SECRET_KEY`, `JWT_ALGORITHM`, and `JWT_EXPIRATION_MINUTES`.
* Added `app/core/jwt_auth.py`.
* Added JWT creation with `sub`, `exp`, and `jti`.
* Added JWT decode and active user resolution.
* Added `POST /auth/login` for Internal Alpha `user_id + dev_token` login.
* Added Dashboard Local Login panel that stores JWT in browser localStorage.
* Added AIHome.law branded frontend product shell.
* Added grouped sidebar navigation and topbar status area.
* Added reusable frontend UI primitives for cards, badges, buttons, section headers, and info rows.
* Added Workspaces and Runtime pages.
* Added modular Dashboard components.

## Changed

* Backend app version is now `3.2.0`.
* `Authorization: Bearer <jwt>` is now the first auth mode checked.
* Bearer dev token, `X-Dev-Token`, and local fallback remain supported.
* `GET /auth/status` now returns `jwt`, `dev_token`, or `local_fallback` and includes `expires_at`.
* Frontend API calls made in the browser now attach the stored JWT.
* `.env.example` now documents JWT local example configuration.
* Frontend API services are grouped by resource modules.
* Core frontend types are centralized in `frontend/types/index.ts`.
* Cases, Reports, Skills, and Case Detail pages now use the AIHome.law legal-tech console styling.

## Notes

* v3.2 is not a complete production login system.
* Current login uses `user_id + dev_token`.
* `JWT_SECRET_KEY` must be replaced in production.
* Local fallback remains local-development only.
* v3.3 can add formal password login, OAuth, or SSO.

## v3.2-B AIHome.law Branded Dashboard

The frontend now presents the internal alpha as `AIHome.law`, with the tagline `AI Workspace for Legal Work`.

The shell is designed for future product expansion:

* Sidebar groups: Primary, Intelligence, Workspace, System.
* Topbar status: workspace, auth mode, runtime provider, user.
* Reserved future items: Experience Packages, Users, Audit Logs, Settings.
* Dashboard modules: BrandHero, AuthStatusCard, WorkspaceSummaryCard, RuntimeStatusCard, StatsGrid, RecentCases, QuickActions.
