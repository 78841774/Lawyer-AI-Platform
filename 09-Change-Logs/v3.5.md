# v3.5 Change Log

## v3.5 Run History / Fact Dedup Foundation

Added runtime history and deterministic fact deduplication for repeated case runtime operations.

### Added

* Added `extraction_runs`, `analysis_runs`, and `report_runs` models.
* Added runtime run repository for run creation, latest marker updates, completion, failure, and history queries.
* Added `GET /cases/{case_id}/runtime-runs`.
* Added `GET /cases/{case_id}/runtime-runs/latest`.
* `POST /cases/{case_id}/facts/extract` now returns run metadata and dedup stats.
* `POST /cases/{case_id}/analysis/run` now returns run metadata, source fact ids, and source refs.
* `POST /cases/{case_id}/reports/generate` now returns run metadata and report source refs.
* Intake Status now returns latest extraction, analysis, and report run ids when available.
* Case Detail now displays `运行历史` with fact extraction, legal analysis, and report generation groups.
* Runtime page notes that Run History is available from Case Detail.
* Added `backend/scripts/validate_run_history_v3_5.py`.

### Fact Dedup

Fact extraction now reuses an existing fact when these values match in the same case:

* normalized fact content
* source material id
* fact type

The response includes:

* `facts_created_count`
* `facts_reused_count`
* `facts_skipped_count`

### Not Included

* No complex OCR.
* No vector or semantic deduplication.
* No DeepSeek Live validation.
* No Skill Training main-chain changes.
* No `/skill-candidates/*` API.
* No production deployment changes.
* No tag created.
