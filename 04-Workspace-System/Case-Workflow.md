# Case Workflow（案件工作流）规范

## 一、定位

Workspace System 的核心是案件管理和能力应用。所有案件处理必须通过标准化流程，确保事实层、法律层和策略层的数据完整、可追踪。

---

## 二、工作流步骤

### 1. 案件接入

- 接收用户或系统提交的案件
- 分配唯一案件 ID
- 确认案件所需能力类型（Fact、Legal、Strategy）

### 2. 材料收集与提取

- 收集案件相关材料
- 使用 Fact Extraction 模块生成事实列表
- 对事实进行分层整理（事实层、证据层、上下文层）

### 3. 法律分析

- 使用 Legal Analysis 模块对事实进行初步分析
- 自动匹配相关法律条文
- 生成分析报告草稿

### 4. 能力调用

- 调用 Experience Package（EP）完成策略设计或报告生成
- 确保调用 Skill 与案件类型匹配
- 记录调用日志

### 5. 结果验证与反馈

- 人工或自动验证分析结果
- 对 EP 调用结果打分（Score）
- 将反馈用于 Skill 优化和 Gate 评估

### 6. 案件归档

- 保存案件完整数据
- 保存分析、策略、EP 调用日志
- 支持后续检索和复用

---

## 三、原则

- 案件处理必须基于事实层完整性
- 所有 EP 调用需遵循 Gate 验证结果
- 结果透明，可追溯
- 案件数据与 Skill 数据分离
