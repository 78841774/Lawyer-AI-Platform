# AIHome.law Pitch Deck Outline v7.9

## 1. 问题背景

律师使用 AI 工具时需要可控、可复核、可追踪。草稿、来源、复核和交付边界如果没有清晰门禁，容易造成误用。

## 2. AIHome.law 定位

AIHome.law 是律师工作流辅助平台，重点是组织受控生产流程，而不是承担律师判断或输出最终结论。

## 3. Personal Production 工作流

展示从材料处理、AI 草稿、信息核验、技能沉淀、受控案件生产到交付包草案的完整链路。

## 4. Story Flow 演示

以 `/personal-showcase-pack` 展示 v7.3-v7.6 能力如何串联为可讲述、可截图、可录屏的 mock metadata 流程。

## 5. Source Trace / Lawyer Review

强调来源可追踪、律师复核必需。每个演示样本都应明确不代表真实案例结果。

## 6. Delivery Packet / Final Lock

展示交付包草案、交付项清单、来源追踪包、导出准备度和 Final Lock。Final Lock 仅表示受控 metadata 锁定，不触发真实导出。

## 7. Safety Boundary

说明 mock-first、provider-gated、metadata-only、draft-only、不生成最终法律意见、不自动对外交付。

## 8. 技术架构

展示前端页面矩阵、FastAPI backend、Personal Production runtime registry、regression suite 和安全检查脚本。

## 9. 当前进展

v7.0-v7.8 已完成 Personal Production runtime foundation、AI Gateway、Material Runtime、Intelligence Gateway、Skill Studio、Case Production、Delivery Packet、Showcase Pack 和 UI Polish。
