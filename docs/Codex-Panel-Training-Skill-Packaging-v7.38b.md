# Codex Panel Training Skill Packaging v7.38b

v7.38b adds a Codex panel / agent Skill at `.agents/skills/lawyer-ai-training-skill/`.

This Skill is for Codex-assisted development and training workflow orchestration. It is not the business-system Training Skill Spec generated by `/personal-skill-studio/training-artifacts/codex-training-skills/*`.

## What It Fixes

The v7.35a-v7.38 business chain already creates controlled training artifacts, redacted experience packages, safe callable Training Skill Spec metadata, and internal training simulation artifacts. Those artifacts live in the application backend and frontend.

v7.38b adds a separate Codex-readable Skill so Codex can consistently load the training workflow rules before future training development, dry-run validation, artifact checking, and safety review.

## Difference From Business Training Skill Spec

- Codex Panel Training Skill: `.agents/skills/lawyer-ai-training-skill/`, used by Codex / agent workflow.
- Business Training Skill Spec: backend metadata artifact, used by the Lawyer-AI-Platform training artifact chain.
- The panel Skill guides development behavior and checks.
- The business Spec describes metadata output, provider aliases, gates, schema, prompt strategy, audit, and source trace.

Neither layer contains real secrets, raw case materials, OCR full text, provider raw responses, or local absolute paths.

## Safety

- No provider calls are added.
- No API key values are read or written.
- No production Skill is published.
- No runtime package is replaced.
- No final legal opinion or formal report is generated.
- All downstream training outputs remain redacted, abstracted, metadata-only, audit-required, and source-trace-required.

## Loading Guidance

For future training-related Codex tasks, explicitly ask Codex to use `lawyer-ai-training-skill`, or ensure Codex reads `.agents/skills/lawyer-ai-training-skill/SKILL.md` before editing training pipeline files.
