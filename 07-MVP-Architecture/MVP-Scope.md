# MVP Scope

MVP Scope 定义 Lawyer AI Platform 第一阶段的功能边界。

第一阶段 MVP 的目标是实现一个可运行的案件工作流闭环：创建案件，上传材料，抽取事实，完成法律分析，生成基础报告。MVP 暂不追求完整训练闭环、复杂协同、全量案件类型覆盖或高级权限体系。

## MVP 第一阶段功能

### 案件管理

支持创建、查询和查看案件。

核心范围：

* 新建案件，填写案件名称、案件类型、当事人和案件目标。
* 案件列表展示案件状态、更新时间、材料数量、事实数量和报告数量。
* 案件详情展示案件基础信息和工作流进度。
* 支持绑定一个 Experience Package。

暂不包含：

* 多团队复杂权限。
* 案件协同审批流。
* 批量案件导入。

### 材料上传

支持上传案件材料并完成基础解析。

核心范围：

* 上传 PDF、图片、Word、TXT 等常见材料。
* 保存材料元数据、原始文件路径和解析状态。
* 对可解析材料生成文本内容或 OCR 结果。
* 支持材料列表、材料详情和材料文本预览。

暂不包含：

* 高级证据目录自动编排。
* 全格式复杂版面还原。
* 大规模文件批处理。

### Fact Extraction

支持从案件材料中抽取结构化事实。

核心范围：

* 基于材料文本生成 Fact Schema。
* 展示 fact_id、description、fact_type、source_materials、evidence_refs、confidence、status。
* 支持人工确认、驳回和争议标记。
* 确保事实可追溯到材料来源。

暂不包含：

* 多轮事实调查策略。
* 复杂事实冲突自动裁判。
* 跨案件事实复用。

### Legal Analysis

支持基于已抽取事实生成法律分析。

核心范围：

* 基于 confirmed 或 verified facts 生成 Legal Reasoning Schema。
* 输出法律争点、关联事实、法律规则、推理链、结论、风险等级和引用。
* 支持律师复核和重新生成。
* 保证法律分析不得脱离 Fact Schema。

暂不包含：

* 全自动类案检索闭环。
* 多法域支持。
* 复杂诉讼策略优化。

### Report Generation

支持生成基础法律分析报告。

核心范围：

* 根据案件事实和法律分析生成 Markdown 报告。
* 报告包含案件概要、事实摘要、法律争点、分析结论和风险提示。
* 支持报告预览、重新生成和下载。
* 报告保留关联 case_id、fact_id、issue_id 和 package_id。

暂不包含：

* 复杂 Word 排版。
* 自动生成全套诉讼文书。
* 多模板批量报告生成。

## MVP 成功标准

* 一个合同纠纷样例案件可以完成端到端流程。
* 每个输出结论都能追溯到事实和材料。
* MVP 服务可以本地部署并完成演示。
* 核心 API 可以被前端稳定调用。
* 报告可以作为律师复核底稿使用。
